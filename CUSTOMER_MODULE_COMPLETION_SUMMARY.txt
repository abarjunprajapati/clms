## 🎯 Main Features

### ⭐ Annexure 3/A Registration Form (THE MAIN FEATURE)
**Two-Tab Interface:**
1. **Basic Info Tab**
   - Auto-filled customer details (from SAP)
   - Auto-filled contractor/vendor details (from work order)

2. **Registration Tab**
   - Work awarding department
   - EPF/ESI/KLWF registration status & exemption reasons
   - Insurance policy details
   - Safety training certificate
   - Gate pass approval

**Workflow:**
```
Fill Form → Save as Draft → Edit → Submit for Review → Under Review → Approved/Rejected
```

## 🔍 Key Differences: Contractor vs Customer

| Aspect | Contractor | Customer |
|--------|-----------|----------|
| **Login ID** | Vendor Code | Customer Code |
| **Master Table** | contractors | sap_customer_master |
| **User Role** | contractor | customer |
| **Registration Form** | annexure-2a.php | annexure-3a.php |
| **Form Storage** | contractors table | customer_annexure3a table |
| **Work Context** | Single entity | Multiple work orders |
| **Primary Focus** | Worker management | Deployment management |

## 📋 Status & Workflow

### Form Status Values
- **draft** - Being edited, can save/edit/submit
- **under_review** - Waiting for officer approval, cannot edit
- **approved** - Approved by officer, completed
- **rejected** - Rejected with reason, can resubmit

### Compliance Status Values
- **pending** - Submitted, awaiting verification
- **submitted** - Officer verifying
- **approved** - Officer approved
- **rejected** - Officer rejected
