## 📋 Form Structure (2 Tabs)

### TAB 1: BASIC INFORMATION
- Customer Code (readonly)
- Customer Name (readonly)
- Email (readonly)
- Mobile (readonly)

### TAB 2: REGISTRATION DETAILS (ALL ANNEXURE 2A FIELDS)

#### 1. **Work Awarding Department** ✓
- Field: `work_awarding_department` (dropdown)
- Source: master_departments table
- Validation: Required

#### 2. **EPF Registration** ✓
- `epf_code` - Text input
- `epf_registered` - Radio (YES/NO)
- `epf_non_registration_reason` - Textarea (if NO selected)

#### 3. **ESI Registration** ✓
- `esi_code` - Text input
- `esi_registered` - Radio (YES/NO)
- `esi_non_registration_reason` - Textarea (if NO selected)

#### 4. **Wage Declaration** ✓
- `wage_declaration` - Checkbox
- Text: "I declare to pay minimum wage as per government norms"

#### 5. **Employee Compensation Policy (ECP)** ✓
- `ecp_covered` - Radio (YES/NO)
- If YES → Dynamic table with:
  - `ecp_number` - Policy number
  - `ecp_valid_from` - Date
  - `ecp_valid_to` - Date
  - `ecp_workers` - Number of workers
  - Add/Remove rows capability
- If NO → `ecp_exemption_reason` - Textarea

#### 6. **Workforce Details** ✓
- `workers_proposed_to_be_engaged` - Number (required)
  - Shows warning if ≥ 20 workers (license mandatory)
- `worker_categories` - Checkboxes (at least one required)
  - Skilled
  - Semi-skilled
  - Unskilled

#### 7. **Labour License Details** ✓
- Dynamic table with:
  - `license_no` - License number
  - `license_validity` - Issued by
  - `issued_date` - Date
  - `expiry_date` - Date
  - Add/Remove rows capability
- Mandatory badge when workers ≥ 20

#### 8. **Additional Details** ✓
- `klwf_registration` - KLWF Registration No (optional)
- `labour_identification_no` - LIN (numeric only, optional)
- `contact_person` - Contact name (required, alphabets only)
- `mobile_primary` - Primary mobile (required, 10 digits)
- `mobile_secondary` - Secondary mobile (optional, 10 digits)
- `remarks` - Additional remarks (optional)

## 🎯 Key Features

### Form Behavior
1. **Draft Mode**: All fields editable
2. **Under Review Mode**: All fields locked (readonly)
3. **Approved/Rejected Mode**: Display only

### Dynamic Sections
- **ECP Table** appears/disappears based on "ECP Covered" selection
- **ECP Reason** appears when "NO" is selected
- **Add/Remove Row buttons** for ECP and License tables

### Validation
- Work Awarding Department: Required
- EPF/ESI: Either YES or both can't be NO
- Wage Declaration: Must be checked
- Worker Categories: At least one required
- License mandatory if workers ≥ 20 (with warning)
- Contact person: Alphabets only
- Mobile: 10 digits pattern
- LIN: Numeric only pattern

### Form Actions
- **Save as Draft** - Saves with status: draft
- **Submit for Review** - Changes status to: under_review
- **Lock on Submit** - Form becomes readonly once submitted

## ✅ Validation Rules

### Field-Level Validation
| Field | Validation | Required |
|-------|-----------|----------|
| Work Department | Dropdown | Yes |
| EPF Code | Text | If EPF=YES |
| EPF Reason | Textarea min 3 chars | If EPF=NO |
| ESI Code | Text | If ESI=YES |
| ESI Reason | Textarea min 3 chars | If ESI=NO |
| Wage Declaration | Checkbox | Yes |
| Worker Count | Numeric ≥ 0 | Yes |
| Categories | At least 1 checkbox | Yes |
| License* | Table rows | If workers ≥ 20 |
| Contact Person | Alphabets only | Yes |
| Primary Mobile | 10 digits | Yes |
| Secondary Mobile | 10 digits | No |

**\* License mandatory when workers_proposed ≥ 20 (with warning)**

### Business Rules
1. Either EPF or ESI must be YES (both cannot be NO)
2. If ECP = YES, must fill policy details
3. If ECP = NO, must provide exemption reason
4. At least one worker category must be selected
5. Wage declaration must be accepted
